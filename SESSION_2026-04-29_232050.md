## Post-mortem: What went wrong 2026-04-29

### Sequence of events

| Time CDT | Event |
|----------|-------|
| Pre-15:10 | Session started with $-2.07 settlement loss (NO-bias, prev window) |
| 15:10–15:30 | 7 trades into 16:00 CDT window near BTC $75,600. Settled profitably. |
| 16:10–19:40 | 46 more trades across 17:00, 18:00, 19:00, 20:00 CDT windows. BTC climbed $75,646 → $76,138. Classifier predominantly calling NO at $75,700–$76,200. |
| 20:00 CDT | **Wipeout.** $7.38 deployed → $1.00 recovered. All NO bets at sub-$76,100 strikes lost as BTC settled ~$76,050. Portfolio $12.05 → $5.67. |
| 20:10+ | Drawdown halt fires (69.5% ≥ 50%) — system stands down. |

### Root causes

**1. Directional blindness / NO-proximity failure**

The model placed NO bets at strikes where BTC was already at or above the level.
Worst examples today:

| Time | BTC spot | NO strike | Gap |
|------|----------|-----------|-----|
| 16:30 | $75,799 | $75,800 | $1 |
| 17:30 | $75,811 | $75,800 | −$11 (BTC above!) |
| 16:40 | $75,898 | $75,900 | $2 |

No guard existed to reject these placements.

**2. Orderbook gate broken since deployment (2026-04-29 15:30)**

The gate imported `fetch_orderbook` and `imbalance_allows_bet` from `orderbook_features`,
but the module resolved to `gemini_trader/orderbook_features.py` (legacy file, different
API). The correct module is `scripts/orderbook_features.py`. The `try/except` caught
the `ImportError` and warned but allowed trades to proceed — the gate was silently dead.

Root cause: `sys.path.insert(0, GEMINI_TRADER_DIR)` put gemini_trader before the
workspace `scripts/` directory, so the wrong file was imported.

**3. RSI using 5-candle simple rolling mean (not a real RSI)**

`w = 5` was used for all rolling computations including RSI. With only 5 candles, five
consecutive up-ticks pin RSI=100 and five down-ticks pin RSI=0. Today's log showed
values swinging 0→100→0 every 10–20 minutes — pure noise. The regime gates based on
RSI (`REGIME_RSI_LOW/HIGH`) were firing on artifacts, not real momentum.

**4. GPU unavailable mid-session (18:10+); CPU fallback silently continued**

Starting at 18:10 CDT, XGBoost fell back to CPU (`No visible GPU found`). The old
code forced `device="cpu"` on the booster and continued inference. Under the GPU
contention model, this run should have been skipped entirely — GPU unavailability means
another process (Ollama, training) owns those resources and the inference clock is
potentially corrupted.

**5. YES bets chronically unprofitable all-time**

All-time: `YES 112W/48L (70%) $-10.81`. Even at 70% win rate, paying 70–90¢ per YES
contract produces negative EV. The classifier's YES edge estimates are miscalibrated.
Not fixed in this session — requires retraining or explicit YES suppression.

## Changes implemented

### Fix 1 — GPU contention model (hard fail, no CPU fallback)

**File:** `scripts/auto_trader.py` — `run_classifier()` block

**Before:** `pick_device()` returned `"cpu"` when GPU was unavailable; code forced
`clf.clf.get_booster().set_param({"device": "cpu"})` and continued inference.

**After:** If `pick_device()` returns `"cpu"`, `run_classifier()` immediately returns
`None`. The `run()` function detects the `None` and logs `[SKIP] Classifier returned
no result` — no trades placed.

```
[SKIP] GPU unavailable (< 1500 MiB free) — standing down (GPU contention model)
```

Threshold: 1500 MiB free on at least one GPU (via `pick_device()` in
`gemini_trader/classifier_device.py`).

**Rationale:** The system's GPUs should be in constant productive use. The classifier
(~15 `predict_proba` calls per tick) does not warrant bumping higher-priority GPU work.
If the GPU is busy, the system stands down rather than trading on potentially stale
or incorrect CPU-side signals.

### Fix 2 — RSI window corrected to 14-period Wilder's smoothing

**File:** `scripts/auto_trader.py` — `run_classifier()`, RSI block

**Before:**
```python
w = 5
gain = delta.where(delta > 0, 0).rolling(w).mean()
loss = (-delta.where(delta < 0, 0)).rolling(w).mean()
df["rsi"] = 100 - (100 / (1 + (gain / loss)))
```

**After:**
```python
RSI_PERIOD = 14
gain = delta.where(delta > 0, 0).ewm(alpha=1/RSI_PERIOD, adjust=False).mean()
loss = (-delta.where(delta < 0, 0)).ewm(alpha=1/RSI_PERIOD, adjust=False).mean()
df["rsi"] = 100 - (100 / (1 + (gain / loss)))
```

Other feature windows (`price_ma`, `price_volatility`, `price_momentum`, `ma_slope`)
remain at `w=5` to preserve classifier training-data alignment. RSI is kept as
a classifier feature — the model will see RSI(14) values going forward. Retraining
with RSI(14) is a follow-up task.

### Fix 3 — Orderbook gate repaired

**File:** `scripts/auto_trader.py` — module-level imports + gate block

**Root cause:** `from orderbook_features import fetch_orderbook, imbalance_allows_bet`
resolved to `gemini_trader/orderbook_features.py` (legacy file without those functions)
because `sys.path.insert(0, GEMINI_TRADER_DIR)` placed gemini_trader before
`scripts/` on the path.

**Fix:** Load `scripts/orderbook_features.py` at module level using an absolute path
via `importlib.util.spec_from_file_location`. The result is stored in module-level
`_ob_mod`; the gate now calls `_ob_mod.fetch_orderbook(...)` and
`_ob_mod.imbalance_allows_bet(...)` directly — no path ambiguity.

Also removed the redundant `sys.path.insert(0, GEMINI_TRADER_DIR)` that was firing
inside `run_classifier()` on every invocation (the module-level insert is sufficient).

### Fix 4 — NO proximity guard

**File:** `scripts/auto_trader.py` — strike evaluation loop
**Service file:** `NO_PROXIMITY_BUFFER=200` added to Environment

**Logic:**
```python
if side == "NO" and current_price >= strike - NO_PROXIMITY_BUFFER:
    # reject — spot too close to or already above the NO strike
    continue
```

A NO bet at strike $X requires BTC to be below $X at settlement. If the current
spot is within $200 of the strike (or above it), the probability of a favorable
outcome is too low to trade — any upward tick closes the gap immediately.

Examples that would now be blocked at `NO_PROXIMITY_BUFFER=200`:

| BTC spot | NO strike | Gap | Action |
|----------|-----------|-----|--------|
| $75,799 | $75,800 | $1 | SKIP |
| $75,811 | $75,800 | −$11 | SKIP |
| $75,898 | $75,900 | $2 | SKIP |
| $75,841 | $75,900 | $59 | SKIP |
| $75,841 | $76,100 | $259 | ALLOW |
| $75,646 | $76,000 | $354 | ALLOW |

Tunable via service `Environment="NO_PROXIMITY_BUFFER=200"` without code changes.
