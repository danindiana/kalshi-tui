## Task 2 — Orderbook feature module scaffolded

**File**: `scripts/orderbook_features.py`

### Public API

```python
fetch_orderbook(signer: KalshiSigner, ticker: str, timeout: float = 5.0)
    → Optional[OrderbookSnapshot]

imbalance_allows_bet(snap: Optional[OrderbookSnapshot], side: str,
                     threshold: float = 0.65)
    → tuple[bool, str]   # (allowed, reason)
```

### OrderbookSnapshot fields

| Field | Type | Meaning |
|-------|------|---------|
| `yes_volume` | int | Total resting YES contracts across all price levels |
| `no_volume` | int | Total resting NO contracts |
| `yes_imbalance` | float | `yes_vol / (yes_vol + no_vol)`; 0.5 = balanced |
| `best_yes_bid` | int | Highest YES bid in cents |
| `best_no_bid` | int | Highest NO bid in cents |
| `market_spread` | int | `100 - best_yes - best_no` cents (lower = tighter) |
| `fetched_at` | float | Unix timestamp of fetch |

### Gate logic

```
NO  bet suppressed when yes_imbalance > threshold (default 0.65)
YES bet suppressed when yes_imbalance < (1 - threshold)
```

Fail-open conditions (gate bypassed, bet allowed):
- `snap is None` — API error / timeout → `ob_unavailable (fail-open)`
- `market_spread > 30¢` — thin book, imbalance is noisy → `ob_thin (gate bypassed)`

### Why hard gate, not Kelly multiplier

Kelly already sizes optimally given model edge vs. market price. The orderbook
imbalance is the *same information* that set the market price — multiplying Kelly
by it would double-count, biasing sizing toward under-betting in thick markets.
What we need is a **veto** for cases where the crowd is heavily positioned against
our model's direction. Hard gate is the correct abstraction.

### API call cost

1 extra GET per candidate strike. With `MAX_PLACEMENTS_PER_RUN=3` and typically
4–10 candidates, this is 4–10 additional requests per run (Kalshi rate limit:
600 req/min — well within budget).

### Change 3 — Gate block (after MACD gate, before Kelly)

```python
        # --- NEW ORDERBOOK CONVICTION GATE ---
        try:
            from orderbook_features import fetch_orderbook, imbalance_allows_bet
            ob_snap = fetch_orderbook(signer, mk["ticker"])
            ob_allowed, ob_reason = imbalance_allows_bet(ob_snap, side, YES_IMBALANCE_THRESHOLD)
            if not ob_allowed:
                print(f"  [SKIP-OB] Strike ${strike:,.0f}: {ob_reason}")
                continue
        except Exception as e:
            print(f"  [WARN] Failed to evaluate orderbook gate for {mk['ticker']}: {e}")
        # --------------------------------
```

The outer `try/except` ensures a broken import or unexpected API shape never
crashes a live run — it degrades to warn-and-continue.
